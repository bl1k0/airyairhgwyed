Database Setup — Run this SQL once in Supabase SQL Editor
-- 1. Create the quotes table CREATE TABLE IF NOT EXISTS public.quotes ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, name text NOT NULL, phone text NOT NULL, email text, service text NOT NULL, location text, message text, status text NOT NULL DEFAULT 'new', created_at timestamptz DEFAULT now() ); -- 2. Enable Row Level Security ALTER TABLE public.quotes ENABLE ROW LEVEL SECURITY; -- 3. Allow anonymous INSERT (form submissions from your website) CREATE POLICY "Allow public insert" ON public.quotes FOR INSERT TO anon WITH CHECK (true); -- 4. BLOCK anonymous SELECT (admin reads via service_role or RLS bypass) -- The admin panel uses your anon key + a server-side function OR -- you can grant select to authenticated only. -- Option A (simplest — recommended): create a Postgres function -- that the admin calls with the anon key, bypassing RLS. -- Run the function below: CREATE OR REPLACE FUNCTION public.get_quotes() RETURNS SETOF public.quotes LANGUAGE sql SECURITY DEFINER -- runs as owner, bypasses RLS AS $$ SELECT * FROM public.quotes ORDER BY created_at DESC; $$; -- Grant execute to anon so the admin panel can call it GRANT EXECUTE ON FUNCTION public.get_quotes() TO anon;
Step-by-step:
1. Go to supabase.com/dashboard → your project
2. Click SQL Editor in the left sidebar
3. Paste the SQL above and click Run
4. Copy your Project URL and anon key from Settings → API
5. Paste both above and click Save & Apply
